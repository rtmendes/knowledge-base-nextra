## Phase 1: Initial Setup (5 minutes)

### Step 1: Create Vercel Account

**What you'll do:** Sign up for a Vercel account to host your application

1.   Go to [https://vercel.com/signup](https://vercel.com/signup)
2.   Click "Continue with GitHub" (What you'll see: GitHub login button prominently displayed)
3.   If you don't have a GitHub account, click "Continue with Email" and enter your email
4.   Follow the prompts to create your account (You'll see: account setup form)
5.   On the Vercel dashboard, click "New Project" (You'll see: a button labeled "New Project" in the upper right)

**Note:** Vercel's free tier is sufficient to start. You'll see plan options, but you can select the Hobby (free) plan.

### Step 2: Create GitHub Repository

**What you'll do:** Create a GitHub repository to store your code

1.   Go to [https://github.com/new](https://github.com/new) (You'll see: "Create a new repository" page)
2.   Enter "youtube-pod-analytics" as Repository name
3.   Set visibility to "Private" (optional)
4.   Check "Add a README file"
5.   Click "Create repository" (You'll see: your new repository page with README)

**Note:** GitHub's free plan allows unlimited private repositories.

### Step 3: Connect GitHub to Vercel

**What you'll do:** Connect your GitHub repository to Vercel for deployment

1.   On Vercel dashboard, click "Add New..." → "Project" (You'll see: a list of your GitHub repositories)
2.   Find and select "youtube-pod-analytics"
3.   Click "Import" (You'll see: project configuration page)
4.   Keep default settings (Framework Preset: Next.js)
5.   Click "Deploy" (You'll see: deployment progress and then success message)

**Success Check:** You'll see "Congratulations!" and a link to your deployed site (but it's just a basic site at this point).

## Phase 2: Template Installation (5 minutes)

### Step 4: Download Project Template

**What you'll do:** Download a pre-built template for your project

1.   Go to [https://vercel.com/templates/next.js/dashboard-nextjs](https://vercel.com/templates/next.js/dashboard-nextjs)
2.   Click "Clone Template" (You'll see: Dialog with deployment options)
3.   Select "GitHub" as Git provider
4.   Repository Name: enter "youtube-pod-analytics"
5.   Click "Create" (You'll see: deployment progress)

**Important:** If you already created a repository with this name in Step 2, you'll need to use a different name here or delete the previous repository.

### Step 5: Customize the Template

**What you'll do:** Make basic customizations to the dashboard template

1.   On GitHub, navigate to your repository (You'll see: repository files)
2.   Click on "app/page.tsx" file (You'll see: code editor for the file)
3.   Click the pencil icon to edit (You'll see: edit mode activated)
4.   Change the title from "Dashboard" to "YouTube POD Analytics"
5.   Commit changes (scroll down, add commit message, click "Commit changes")

**Note:** Vercel will automatically redeploy your site with these changes.

## Phase 3: API Configuration (5 minutes)

### Step 6: Set Up Google Cloud Project

**What you'll do:** Create a Google Cloud project for YouTube API access

1.   Go to [https://console.cloud.google.com/](https://console.cloud.google.com/)
2.   Click "Select a project" at the top (You'll see: project selection dialog)
3.   Click "NEW PROJECT" (You'll see: new project creation form)
4.   Enter "YouTube POD Analytics" as Project name
5.   Click "CREATE" (You'll see: project dashboard after creation)

### Step 7: Enable YouTube Data API

**What you'll do:** Enable the YouTube Data API for your project

1.   In Google Cloud Console, click "APIs & Services" → "Library" in the left menu
2.   Search for "YouTube Data API v3" (You'll see: API card in results)
3.   Click on "YouTube Data API v3" (You'll see: API overview page)
4.   Click "ENABLE" (You'll see: API details page after enabling)
5.   Click "CREATE CREDENTIALS" (You'll see: credential creation wizard)

### Step 8: Create API Credentials

**What you'll do:** Generate API credentials for YouTube API access

1.   On the "Create credentials" page, select: 
    *   Which API are you using? → "YouTube Data API v3"
    *   Where will you be calling the API from? → "Web server"
    *   What data will you be accessing? → "User data"

2.   Click "NEXT" (You'll see: OAuth consent screen setup)
3.   Select "External" user type, click "CREATE" (You'll see: OAuth consent form)
4.   Fill in required fields: 
    *   App name: "YouTube POD Analytics"
    *   User support email: Your email
    *   Developer contact information: Your email

5.   Click "SAVE AND CONTINUE" through the remaining screens (Scopes, Test users)
6.   Click "BACK TO DASHBOARD" (You'll see: OAuth consent screen dashboard)

**Note:** For development, you'll be in "Testing" mode, which limits access to specific test users you add.

### Step 9: Create OAuth Client ID

**What you'll do:** Generate OAuth credentials for user authentication

1.   Go to "Credentials" tab in the left menu (You'll see: credentials dashboard)
2.   Click "+ CREATE CREDENTIALS" and select "OAuth client ID" (You'll see: client ID creation form)
3.   Application type: select "Web application"
4.   Name: "YouTube POD Analytics Web Client"
5.   Authorized JavaScript origins: Add your Vercel app URL (format: https://youtube-pod-analytics.vercel.app)
6.   Authorized redirect URIs: Add your Vercel app URL + "/api/auth/callback/google" (format: https://youtube-pod-analytics.vercel.app/api/auth/callback/google)
7.   Click "CREATE" (You'll see: client ID and secret in a popup)
8.   Copy the Client ID and Client Secret to a secure temporary location (you'll need these shortly)

**Important:** Keep your Client Secret secure and never expose it publicly.

## Phase 4: Environment Configuration (3 minutes)

### Step 10: Configure Environment Variables in Vercel

**What you'll do:** Add your API credentials to Vercel as environment variables

1.   Go to your Vercel dashboard and select your project (You'll see: project overview)
2.   Click on "Settings" tab (You'll see: project settings)
3.   Click on "Environment Variables" in the left menu (You'll see: environment variables page)
4.   Add the following variables one by one: 
    *   Key: GOOGLE_CLIENT_ID | Value: [Your Client ID from Step 9]
    *   Key: GOOGLE_CLIENT_SECRET | Value: [Your Client Secret from Step 9]
    *   Key: NEXTAUTH_SECRET | Value: [Generate a random string or use "your-nextauth-secret" for testing]
    *   Key: NEXTAUTH_URL | Value: [Your Vercel app URL, e.g., https://youtube-pod-analytics.vercel.app]

5.   Click "Save" after adding each variable (You'll see: confirmation message)

### Step 11: Redeploy Your Application

**What you'll do:** Trigger a redeployment to apply the environment variables

1.   In Vercel, go to the "Deployments" tab (You'll see: deployment history)
2.   Find your latest deployment and click the three dots menu (You'll see: dropdown menu)
3.   Select "Redeploy" (You'll see: confirmation dialog)
4.   Click "Redeploy" again (You'll see: deployment progress)
5.   Wait for deployment to complete (You'll see: success message with "Deployed successfully")

**Success Check:** Your deployment should complete without errors, and the environment variables will be applied.

## Phase 5: Basic Customization (2 minutes)

### Step 12: Update Dashboard Content

**What you'll do:** Customize the dashboard to match our design

1.   On GitHub, navigate to the repository files (You'll see: file list)
2.   Go to the "app" folder and then open "dashboard/page.tsx" (You'll see: code editor)
3.   Click the pencil icon to edit (You'll see: edit mode)
4.   Replace the "Dashboard" title with "YouTube POD Analytics"
5.   Add a descriptive subtitle like "Discover print-on-demand opportunities from YouTube comments"
6.   Commit changes (scroll down, add commit message, click "Commit changes")

**Note:** These are just basic text changes. For comprehensive customization, refer to the Technical Specification document.

## Phase 6: Testing and Next Steps

### Step 13: Test Your Deployment

**What you'll do:** Visit your deployed application and test it

1.   Go to your Vercel dashboard and select your project (You'll see: project overview)
2.   Click on the deployed URL (format: https://youtube-pod-analytics.vercel.app) (You'll see: your deployed application)
3.   Verify that the site loads correctly (You'll see: dashboard template with your customized title)
4.   Test navigation to different sections (You'll see: different template pages)

**Success Check:** Your application should load without errors and show the basic template structure with your customizations.
