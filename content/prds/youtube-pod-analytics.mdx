# YouTube Pod Analytics — PRD

## Overview & Business Goal

YouTube Pod Analytics is a Next.js SaaS application that analyzes YouTube podcast channel data to surface actionable insights: top-performing episode patterns, optimal posting times, engagement trends, title formulas, and audience growth metrics.

**Target User:** Podcast creators who publish to YouTube (a rapidly growing segment — 1M+ channels), content strategists, and podcast production agencies managing multiple shows.

**Primary Goal:** $200/day ($6,000/month) ARR at target, through subscription-based access to the analytics dashboard.

**Critical Pre-Launch Requirement:** There is an open CVE security vulnerability in the codebase that must be patched before any user-facing launch or marketing.

## Implementation Plan

### Phase 1 — Security Fix + Launch-Ready (This Week, Days 1–7)

**PRIORITY 0 — Fix CVE Before Anything Else**

1. **Security Audit & CVE Fix**
   - Run `npm audit` to identify all known vulnerabilities
   - Run `npx audit-ci --high` to surface high/critical CVEs
   - Identify the specific CVE (likely in a dependency — check Next.js version, React, or API-related packages)
   - Common culprits: outdated `next` version, `axios`, `jsonwebtoken`, `node-fetch`
   - Apply fix: `npm audit fix` for auto-fixable, manual version bump for manual fixes
   - Verify fix: re-run `npm audit` — confirm zero high/critical vulnerabilities
   - Document the CVE: what it was, what it allowed, how it was patched
   - PopeBot can execute this full audit via code analysis

2. **Codebase Audit**
   - Review all API routes for authentication gaps
   - Check for exposed API keys in code (run: `git log --all --full-history -- '*.env'`)
   - Verify YouTube API key is not committed to repo
   - Confirm `.env.local` is in `.gitignore`
   - Review Next.js config for security headers

3. **Core Feature Verification**
   - Test YouTube data pull: does the app successfully fetch channel data?
   - Verify all analytics views render correctly
   - Document what's working vs. broken
   - List missing features against PRD scope

4. **Authentication Setup (if not built)**
   - Implement NextAuth.js with Google OAuth (fits YouTube audience)
   - User table: id, email, plan, stripe_customer_id, channels[]
   - Protect all dashboard routes with session check

5. **Billing Integration**
   - Integrate Stripe Checkout for subscription plans
   - Webhook: handle subscription created/canceled/updated
   - Enforce plan limits in API routes (channel count, history depth)

6. **Production Environment**
   - Deploy to Vercel (connect `rtmendes/youtube-pod-analytics` repo)

## Technical Specifications

### Stack
- **Framework:** Next.js 14+ (App Router)
- **Language:** TypeScript
- **Database:** Supabase (PostgreSQL) or PlanetScale
- **Auth:** NextAuth.js v5
- **Payments:** Stripe
- **Deployment:** Vercel
- **YouTube Data:** YouTube Data API v3

### YouTube API Endpoints Used
- `channels.list` — channel stats and metadata
- `videos.list` — episode-level performance data
- `analytics.query` — views, watch time, impressions (requires OAuth)

### Security Requirements (post-CVE fix)
- All API routes must validate session before returning data
- Rate limiting on all public endpoints (Upstash Redis or Vercel Edge Config)
- HTTPS-only (Vercel default)
- No secrets in client-side code
- Content Security Policy headers
- OWASP Top 10 compliance checklist

### CVE Investigation Guide
```bash
# Run these in the repo to identify vulnerabilities
npm audit --json > audit-report.json
npx better-npm-audit audit -l high
# Check Next.js version
cat package.json | grep '"next"'
# Common fix:
npm install next@latest
npm audit fix
```
